<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin -> script | Untrusted JSON input from Claude Code. Validated by Zod in Plan 01. |
| script -> Perplexity API | Outbound HTTPS call with bearer token auth. API key must not leak to logs. |
| Perplexity API -> script | Untrusted response data. Titles and URLs must be XML-escaped before output. |
| script -> stdout | Output consumed by Claude Code. Must be valid XML only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | API key in Perplexity client | mitigate | API key passed directly to SDK constructor, never logged; getApiKey logs source only at debug level without value |
| T-01-06 | Tampering | Perplexity response data | mitigate | All title/URL values from API response passed through escapeXml() via formatSearchResults |
| T-01-07 | Denial of Service | Perplexity API unreachable | accept | Script exits with non-zero code and clear error message. Retry logic is Phase 2 (SRCH-07). |
| T-01-08 | Information Disclosure | Error messages to stdout | mitigate | All error handling writes to stderr via logger.error; stdout reserved for XML results only |
</threat_model>
