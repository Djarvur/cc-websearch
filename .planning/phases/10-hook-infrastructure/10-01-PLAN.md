## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hook command (inline echo) to Claude Code runtime | Static text input; no untrusted data crosses this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-01 | Spoofing | Hook deny JSON output | LOW | accept | Deny JSON is static text with no user-controlled content. Shell injection via tool names is not possible — matcher is a hardcoded string value in hooks.json, not runtime input. |
| T-10-02 | Tampering | `hooks/hooks.json` file | LOW | accept | File is checked into the repo and distributed with the plugin. If an attacker modifies the hooks file, the tool denial behavior could be removed. Standard git integrity applies. |
| T-10-03 | Information Disclosure | Hook denial reason | LOW | accept | Denial reason text reveals plugin skill names to the model. This is intentional — the redirect is the purpose of the hook. No secrets or credentials are exposed. |
| T-10-04 | Denial of Service | Hook misconfiguration | LOW | mitigate | If hook JSON is malformed (shell escaping error), the hook fails silently (non-zero exit) and the built-in tool proceeds normally. This is the safe failure mode. Verify JSON validity with python3 -m json.tool. |
| T-10-SC | Tampering | npm/pip/cargo installs | LOW | accept | No new packages are installed in Phase 10. Package manager integrity is not in scope. |
</threat_model>
