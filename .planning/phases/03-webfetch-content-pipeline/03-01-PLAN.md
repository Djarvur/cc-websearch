## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `src/lib/fetch.ts` normalizeUrl | mitigate | URL constructor validates scheme; only http/https accepted (Zod `.url()` validates before normalizeUrl runs) |
| T-03-02 | Tampering | `src/lib/fetch.ts` Content-Type check | mitigate | D-12/D-15: whitelist `text/html` and `application/xhtml` only; reject all other Content-Types before processing body |
| T-03-03 | Spoofing | `src/lib/fetch.ts` redirect handling | mitigate | D-10: cross-host redirects NOT followed -- hostname comparison prevents open redirect abuse |
| T-03-04 | Denial of Service | `src/lib/fetch.ts` redirect loop | mitigate | D-11: 10-hop cap prevents infinite redirect loops |
| T-03-05 | Tampering | `src/lib/fetch.ts` HTTP error responses | mitigate | D-14: 4xx/5xx responses rejected immediately; no body extraction for error pages |
| T-03-06 | Tampering | `src/lib/input.ts` schema validation | mitigate | Zod strictObject validation on stdin input; rejects extra fields, validates url format, requires prompt |
| T-03-SC | Tampering | npm install (new packages) | accept | No new packages installed in this plan (native fetch only, all libs already installed) |
</threat_model>
