## Phase Goal

**As a** Claude Code user, **I want to** filter search results by allowed or blocked domains, **so that** I can control which sources appear in my search results.

<objective>

Purpose: Enable domain-level control over search results matching Claude Code's interface exactly.
Output: Domain filter module, input validation extension, wired websearch.ts with domain filtering, rebuilt bundle, passing tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin domain arrays -> filter | Untrusted domain strings from Claude Code input |
| filter output -> Perplexity API | Domain strings passed as API parameter |
| filter output -> DDG post-filter | Domain strings used for URL matching |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Domain strings in allowed/blocked_domains | mitigate | Aggressive normalization (D-13): strip protocol, path, www, trailing slash, lowercase. Only bare domain names reach Perplexity API. New URL() parsing rejects malformed URLs. |
| T-02-04 | Tampering | Perplexity search_domain_filter injection | mitigate | normalizeDomain strips all non-domain components. buildPerplexityDomainFilter caps at 20 domains. No user-controlled strings reach API without normalization. |
</threat_model>
