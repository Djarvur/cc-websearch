## Phase Goal

**As a** Claude Code user, **I want to** search the web even when Perplexity is unavailable, **so that** I always get search results regardless of API status.

<objective>
Implement DDG fallback provider, retry with exponential backoff, and two-tier fallback orchestration. After this plan, WebSearch works end-to-end with either Perplexity or DDG, handles rate limits gracefully, and reports which provider served results.

Purpose: Deliver the core resilience promise -- WebSearch never fails silently and always tries an alternative provider.
Output: Working DDG provider, retry module, rewired websearch.ts with fallback chain, rebuilt bundle, passing tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin -> input parser | Untrusted JSON input from Claude Code |
| WebSearch -> Perplexity API | External API call with API key |
| WebSearch -> DDG Lite | External HTTP request, no authentication |
| retry module -> provider | Internal retry loop wrapping external calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Domain strings in filter input | mitigate | Aggressive normalization in filter.ts (D-13): strip protocol, path, special chars. Perplexity API validates search_domain_filter server-side. |
| T-02-02 | Denial of Service | Retry loop in retry.ts | mitigate | Hard cap on retry count (default 4, max configurable via RETRY_MAX_RETRIES), max delay cap (16s), per-request timeout (30s via RETRY_TIMEOUT). No infinite loops possible. |
| T-02-03 | Information Disclosure | Error messages on stderr | mitigate | Error messages include provider name and error type (D-19) but NOT API keys, request bodies, or sensitive headers. Logger outputs to stderr only, never stdout. |
| T-02-SC | Tampering | npm install (duck-duck-scrape) | mitigate | Legitimacy checkpoint (Task 0) with blocking human verification. Package verified on npm registry and GitHub before install. |
</threat_model>
