    Implement two fixes:

    **1a. Fix withTimeout in `src/lib/retry.ts` (D-16):**
    The current `withTimeout` function races a promise against a timeout. It appears structurally correct but may not handle the case where `fn()` itself hangs forever after the timeout fires (e.g., the promise is never settled). The fix: ensure the timeout error message is recognizable by `isDDGTransientError` (which checks for `ECONNREFUSED|ETIMEDOUT|ENOTFOUND|429|503`) so that timeout-triggered aborts get retried. The timeout error message should include "ETIMEDOUT" as a substring.

    Specifically, change the rejection message from:
    ```
    `Operation timed out after ${ms}ms`
    ```
    to:
    ```
    `Request timed out after ${ms}ms (ETIMEDOUT)`
    ```

    This makes the timeout error match `isDDGTransientError`, so timeouts get proper retry behavior instead of being treated as permanent failures.

    Also consider: if the bug is that withTimeout never triggers because of a promise microtask ordering issue, the fix may need to be different. Read the code carefully, understand the actual bug, and apply the correct fix that makes withTimeout functionally abort retries after timeout. Update the test file to verify the fix.

    **1b. Fix readStdin in `src/lib/input.ts` (D-17):**
    The issue: when stdin is closed without emitting any data (e.g., `echo -n '' | node script.js`), the `process.stdin` async iterator completes immediately, `Buffer.concat(chunks)` produces an empty buffer, and `.trim()` produces `""`. The check `if (!raw)` throws `new Error('No input received on stdin. Provide JSON input via pipe.')`.

    The actual crash scenario: when stdin is not provided at all (e.g., `node script.js` with no pipe), the process may hang waiting for stdin and then crash with a different error. The fix: ensure the function definitively handles the case where stdin is empty OR not piped. If `process.stdin.isTTY` is true (no pipe), throw immediately with a clearer message about stdin being required.

    Add to the function (before the loop):
    ```typescript
    if (process.stdin.isTTY) {
      throw new Error('No input provided. Pipe JSON to stdin or use --query / --url flags.');
    }
    ```

    Make sure the tests account for this new code path. `isTTY` is `undefined` in test environments (not a terminal), so the new code will not throw in existing tests.
