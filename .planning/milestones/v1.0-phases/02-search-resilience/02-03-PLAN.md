## Phase Goal

**As a** Claude Code user, **I want to** get the most results possible even during partial failures, **so that** I never lose useful data when a provider partially succeeds.

Purpose: Maximize result yield during partial failures and provide actionable error messages for debugging.
Output: Updated websearch.ts with merge logic, URL deduplication, detailed errors. Rebuilt bundle. Passing tests.
</objective>

To preserve partial results, change the Perplexity call to a try/catch inside the retry wrapper: attempt to call search(), and if the search itself succeeds but the results are partial (fewer than expected), capture them. In the try block for Perplexity, after a successful search call, set pplxPartial to the results. The retryWithBackoff wrapper will retry on failure. If the final attempt fails (after retries), pplxPartial may still hold results from a prior successful attempt if the failure was on a later retry.

    Alternative (simpler): the retry wrapper retries the ENTIRE function. If Perplexity returns results on attempt 1, then 429 on attempt 2, retryWithBackoff will keep retrying and eventually throw. The results from attempt 1 are lost because retryWithBackoff only returns on success or throws on exhaustion.

    Best approach per RESEARCH.md Pitfall 5: Have the retry wrapper capture partial results. Add a `onPartialResult?: (result: T) => void` callback to retryWithBackoff that fires on each successful attempt before potential retry. There is no "partial results then failure" within a single retryWithBackoff call because success returns immediately.

    The practical interpretation: D-17 covers the edge case where the Perplexity API call partially succeeds (returns data but also signals an error). For the Perplexity SDK, this would be unusual.     let pplxResults: SearchResult[] = [];
    ```
    try {
      const pplxResult = await retryWithBackoff(() => search(query, domainFilter), isTransientError);
      pplxResults = pplxResult.results;
    } catch (pplxErr) {
      // pplxResults may be empty (no partial results captured before failure)
      logger.debug(`Perplexity failed, falling back to DDG: ${(pplxErr as Error).message}`);
      const ddgResults = await retryWithBackoff(() => searchDDG(query), isDDGTransientError);
      const filteredDdg = filterByDomains(ddgResults, parsed.allowed_domains, parsed.blocked_domains);
      // Merge: Perplexity partial (if any) first, then DDG
      results = dedupeAndMerge(pplxResults, filteredDdg);
      provider = pplxResults.length > 0 ? 'perplexity+duckduckgo' : 'duckduckgo';
    }
    ```
